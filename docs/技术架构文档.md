# Deep Research Agent — 技术架构设计文档

## 1. 系统架构

```
用户
  │
  ├─ Streamlit Web UI ────→ FastAPI Server ──→ Task Manager
  │                              │                    │
  └─ API Client (curl/httpx) ────┘                    │
                                                      v
                                           LangGraph Research Workflow
                                              │
                 ┌────────────────────────────┼────────────────────────────┐
                 v                            v                            v
           External Services           Vector Database             LLM Provider
           ┌──────────────┐           ┌──────────────┐           ┌──────────────┐
           │ Search API   │           │   Qdrant     │           │   OpenAI     │

           │ Serper/Bing  │           │              │           │   etc.       │
           └──────────────┘           └──────────────┘           └──────────────┘
```

## 2. LangGraph 工作流

```
START
  │
  v
planner_node ──── 分解研究问题为子问题，生成研究计划
  │
  v
retriever_node ──── 多关键词搜索，收集网页资料
  │
  v
content_extractor_node ──── 抓取网页正文，内容清洗
  │
  v
source_evaluator_node ──── 评估信源质量，筛选合格资料
  │
  v
evidence_builder_node ──── 文本切片 → Embedding → Qdrant 入库
  │
  v
analyst_node ──── RAG 检索证据，生成子问题分析
  │
  v
fact_checker_node ──── 检查证据充分性，判断是否需要补充检索
  │
  ├── 证据不足且未达最大轮数 ──→ retriever_node（下一轮）
  │
  └── 证据充分或达到最大轮数 ──→ report_writer_node
                                    │
                                    v
                                  END（返回最终报告）
```

## 4. 数据流

```
用户输入
  │
  v
ResearchTaskCreate (Pydantic 校验)
  │
  v
ResearchTask (含 task_id)
  │
  v
ResearchState (LangGraph 状态)
  │
  ├── research_plan: dict
  ├── search_results: list[SearchResult]
  ├── crawled_documents: list[CrawledDocument]
  ├── evaluated_sources: list[SourceEvaluation]
  ├── evidence_chunks: list[EvidenceChunk]
  ├── sub_answers: list[SubAnswer]
  ├── fact_check_result: FactCheckResult
  └── final_report: str (Markdown)
  │
  v
TaskService.update_task() → JSON 持久化
  │
  v
API Response → User
```

## 5. 关键设计决策

### 5.1 异步架构
- 所有外部 I/O 使用 `asyncio` + `httpx`
- FastAPI 原生异步支持
- Qdrant 使用 `AsyncQdrantClient`

### 5.2 多轮研究
- `fact_checker_node` 判断是否需要补充搜索
- `should_continue_research()` 条件路由
- 可通过 `max_rounds` 参数控制深度

### 5.3 LLM 调用
- 使用 `LiteLLM` 统一接口，可切换任意模型提供商
- 支持 `response_format` 获取结构化 JSON 输出
- 失败自动重试（tenacity）

### 5.4 可扩展性
- 搜索引擎通过 Provider 模式可扩展
- 向量数据库可替换为其他引擎
- 报告支持 Markdown / JSON 两种格式
- LangGraph 节点可独立替换
